# Работа с удалёнными репозиториями
## Публичные и приватные репозитории на GitHub
**Публичный репозиторий** - репозиторий на сайте [GitHub](https://github.com), который доступен всем пользователям, каждый может его скачать, изучить и редактировать у себя на компьютере.
**Приватный репозиторий** - репозиторий на сайте [GitHub](https://github.com), который можете видеть только Вы и никто другой.

## Создание удалённого репозитория
Для создания удалённого репозитория на [GitHub](https://github.com) необходимо проделаnm следующее:

1. Регистрируемся на [GitHub](https://github.com)
2. Заходим в свой профиль
3. Нажимаем на вкладку **`Repositories`**
4. Нажимаем на кнопку **`New`**
5. В графе **`Repository name`** вводим любое название
    * чтобы потом не запутаться можно назвать также как и файл из первого пункта
6. В графе **`Description`** можно написать описание репозитория, но это необязательно
7. Выбираем, каким мы хотим сделать данный репозиторий, **Приватным** или **Публичным**
8. Нажимаем кнопку **`Create repository`**

## Импортирование файла в удалённый репозиторий
Чтобы импортировать файл в удалённый репозиторий на [GitHub](https://github.com) необходимо воспользоваться подсказками, которые появляются сразу после создания удалённого репозитория. Существует 4 способа:

### 1 способ
Воспользоваться возможностью быстрой настройки. Данный способ сайт предлагает использовать тем пользователям, которые ранее уже делали это.
Для этого необходимо произвести следующие действия:
1. В терминале с папкой-репозиторием ввести команду *git branch -M master* (в некоторых случаях **master** необходимо заменить на **main**).
2. Скопировать ссылку из раздела **`Quick setup`** на [GitHub](https://github.com).
3. В терминале с папкой-репозиторием ввести команду *git remote add origin <ссылка на ваш удалённый репозиторий>*.
4. В терминале с папкой-репозиторием ввести команду *git push origin master* (в некоторых случаях **master** необходимо заменить на **main**).
5. На странице с созданым удалённым репозиторием нажать на кнопку **`Обновить страницу`**.

### 2 способ
Создать новый репозиторий при помощи командной строки. Для этого необходимо ввести следующие команды:
* echo "# <название удалённого репозитория>" >> README.md
* git init
* git add README.md
* git commit -m "first commit"
* git branch -M master (в некоторых случаях **master** необходимо заменить на **main**)
* git remote add origin <ссылка на ваш удалённый репозиторий>
* git push -u origin master (в некоторых случаях **master** необходимо заменить на **main**)

### 3 способ
Вставить существующий репозиторий из командной строки. Для этого необходимо ввести следующие команды:
* git remote add origin <ссылка на ваш удалённый репозиторий>
* git branch -M main
* git push -u origin main

### 4 способ
Импортировать код из другого репозитория. Вы можете инициализировать этот репозиторий кодом из проекта Subversion, Mercurial или TFS. Для этого необходимо нажать на кнопку **`Import code`**.

## Работа с удалённым репозиторием на сайте GitHub
### Редактирование
Удалённым репозиторий можно редактировать. Для этого необходимо перейти в свой профиль на [GitHub](https://github.com), далее во вкладку **`Repositories`** выбрать необходимый репозиторий и нажать на значок карандаша **`Edit this file`**.

После внесения изменений необходимо спуститься вниз той самой страницы в раздел **"Commit changes"** и ввести сообщение к коммиту. Затем нажать на кнопку **`Commit changes`**.

### Экспорт изменений, внесённых в удалённый репозиторий, в локальный репозиторий
Для экспорта изменений, внесённых в удалённый репозиторий, в локальный репозиторий используется команда *git pull origin master*. Для этого в терминале с папкой-репозиторием пишем *git pull origin master*.

### Экспорт изменений, внесённых в локальном репозитории, в удалённый репозиторий
Для экспорта изменений, внесённых в локальном репозитории, в удалённый репозиторий используется команда *git push origin master*. Для этого в терминале с папкой-репозиторием пишем *git push origin master*. Если в терминале высвечивается ошибка, то следует сначала импортировать последнюю версию из удалённого репозитория, при возникновении конфликтов их можно решить одним из ранее описанных способов.

## Работа с чужими репозиториями на Git
Есть несколько вариантов, как можно открыть чужой репозиторий, размещённый на [GitHub](https://github.com). Основным условием для этого является публичность репозитория.

### 1 способ
Найдя публичный репозиторий на сайте [GitHub](https://github.com), необходимо нажать на зелёную кнопку **`<> Code`**, далее в разделе **HTTPS** нажать на кнопку **`Download ZIP`**. После  этого на ваш компьютер скачается файл в формате **.zip**.

Однако у данного способа есть один значительный недостаток, Вы не сможете работать с данным файлом, только просматривать его содержимое.

### 2 способ
Найдя публичный репозиторий на сайте GitHub, необходимо нажать на зелёную кнопку **`<> Code`**, далее в разделе **HTTPS** нужно скопировать ссылку, под которой написано:

**Use Git or checkout with SVN using the web URL.**

Далее необходимо открыть *Visual Studio Code* и там открыть новую пустую папку. После этого используется команда *git clone*. Для её использования нужно в терминале написать *git clone <скопированная ссылка на репозиторий>*. Репозиторий скачается на ваш компьютер.

Чтобы начать работать со скачанным файлом необходимо в терминале написать команду *cd <название скачанного файла>*.

После внесения всех изменений в локальный файл Вы можете предложить внести данные изменения автору репозитория. Для этого необходимо выполнить следующие действия:

1. Открыть страничку данного репозитория на [GitHub](https://github.com) и нажать на кнопку **`Fork`**, а затем на кнопку **`Create fork`**
2. Вносите все необходимые изменения
3. Нажимаете на кнопку **`Pull requests`**
4. Выбираете, в какой репозиторий и в какую его ветку вы хотели бы внети изменения
5. Выбираете, откуда и из какой ветки вы хотели бы внести изменения
6. Нажимаете кнопку **`Create pull request`**
7. Добавляете информацию внесённых изменениях и снова нажимаете кнопку **`Create pull request`**
